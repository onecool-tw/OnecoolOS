# Onecool OS v0.3.0-beta

Onecool OS v0.3.0-beta is the first production-quality Beta release prepared
for the Onecool OS platform and Collectible Radar product workflow.

This release freezes the deterministic local-file based Collectible Radar
pipeline. It does not create a Git tag yet; tagging is deferred until final
human approval.

## Release Overview

v0.3.0-beta validates Onecool OS as a layered personal operating system for
assets, valuation records, deterministic intelligence, review workflows, and
future OFAI context.

The release focuses on Collectible Radar, the first product built on Onecool
OS. Collectible Radar supports safe ingestion from local user-approved files,
preserves source identity, maps observations into valuation history, evaluates
source agreement, produces deterministic intelligence, and assembles
presentation and review outputs.

## Known Limitations

- Live APIs are not implemented.
- Multi-currency source agreement is not implemented.
- Scheduling is not implemented.
- Persistent audit storage is not implemented.
- Authentication and credential management are not implemented.
- PWCC, Goldin, and Fanatics manual imports are pending.
- OCR and image recognition are out of scope.
- LLM recommendations are out of scope.
- Dashboard remains structured-data oriented; no web UI is included.

## Upgrade Notes

- No database migration is required for this release.
- No private user data should be committed.
- Existing local JSON and CSV imports remain local-file based.
- `MarketIntelligence` remains backward compatible without
  `SourceAgreementResult`.
- `SourceAgreementResult` should be supplied by new Collectible Radar flows
  when agreement calculation is needed.
- eBay Sold remains the sports-card Primary Market Price source.
- Card Ladder, Manual, PWCC, Goldin, and Fanatics remain Validation Sources.
