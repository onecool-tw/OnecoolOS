# Onecool OS v0.2.0 Beta Architecture

Onecool OS v0.2.0 Beta freezes the Business Intelligence and Decision
Platform architecture before Decision Engine implementation begins.

## Completed Modules

- Business Logic foundation
- Cash Flow Engine
- Allocation Engine
- Risk Engine
- Performance Engine
- Business Logic Pipeline Runner
- Analytics Integration
- Dashboard Analytics View foundation
- Scenario Engine foundation
- OFAI Foundation
- Decision Engine Architecture Proposal

## Business Intelligence Summary

Business Logic owns deterministic calculations and signals. Analytics owns
derived snapshots. Dashboard owns display-only views. The Business Logic
Pipeline Runner executes registered engines in deterministic order, and
Analytics Integration maps pipeline outputs into Analytics-compatible payloads.

Completed Business Intelligence capabilities:

- Cash flow metrics
- Allocation metrics
- Risk signals and risk metrics
- Unrealized performance metrics
- Pipeline execution reports
- Analytics mapping
- Dashboard analytics presentation

## Scenario Engine Summary

Scenario Engine owns structured scenario objects. It supports deterministic
A/B/C/D planning:

- A: Base Case
- B: Upside Case
- C: Downside Case
- D: Stress Case

Scenario Engine does not perform AI reasoning, market prediction, or
recommendations. It prepares trusted scenario inputs for OFAI.

## OFAI Foundation Summary

OFAI is the Onecool Financial AI orchestration layer. It is not an AI model.
It consumes deterministic outputs from Business Logic, Analytics, Dashboard,
and Scenario Engine to prepare structured decision context.

OFAI does not call LLMs, make investment recommendations, predict markets, or
mutate source data in v0.2.0 Beta.

## Decision Architecture Summary

Decision Engine is approved as the next deterministic layer. It will consume
Business Logic results, Analytics snapshots, Dashboard summaries, ScenarioSet,
and OFAIContext. It will produce decision options, trade-off summaries,
readiness states, scores, and audit trails.

Decision Engine must not:

- call LLMs
- make final user decisions
- predict markets
- bypass deterministic layers
- mutate source data
- present legal, tax, or financial advice as final truth

## Known Limitations

- Decision Engine code is not implemented yet.
- Recommendation Engine is not implemented yet.
- LLM integration is not implemented yet.
- No market prediction is implemented.
- No legal, tax, or financial advice engine is implemented.
- Dashboard remains JSON/display-model oriented; no web UI or charts yet.
- Automation workflows remain future work.
